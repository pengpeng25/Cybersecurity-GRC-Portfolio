## Why Governance Matters at CGT

CGT is a leader in gaming software with a history of innovation — but cybersecurity has been treated as an IT function rather than a business priority. The absence of governance means:

- No executive accountability for security decisions
- No alignment between security investments and business strategy
- No metrics or reporting for leadership decision-making
- No formal oversight of the CISO or cybersecurity program

**Governance transforms cybersecurity from a technical cost center into a strategic business function.**

## Governance Framework: ITGI Approach

The governance model is based on the **IT Governance Institute (ITGI)** framework, which defines cybersecurity governance as a leadership responsibility encompassing:

| ITGI Principle | Description | CGT Application |
|---|---|---|
| **Strategic Alignment** | Security strategy supports business objectives | Protect IP and development capability that drives CGT's revenue |
| **Risk Management** | Risks are identified, assessed, and managed systematically | Formal risk register, risk appetite definitions, treatment plans |
| **Resource Management** | Security resources are allocated efficiently | 6-person team (Phase 1) with cross-functional sourcing to optimize cost |
| **Performance Measurement** | Security effectiveness is measured and reported | KPIs, dashboards, quarterly reporting to executive team |
| **Value Delivery** | Security investments deliver measurable business value | Reduced risk exposure, compliance readiness, IP protection |

## Desired Outcomes

| Outcome | Measurement |
|---|---|
| **Align cybersecurity with CGT's innovation and business goals** | Security controls integrated into SDLC; no security-related product delays |
| **Manage and reduce risks to intellectual property and customer data** | Risk register maintained; red-zone risks reduced by >40% in Year 1 |
| **Ensure efficient resource use and budget oversight** | Security budget tracked against risk reduction metrics |
| **Provide meaningful metrics for leadership decision-making** | Quarterly executive dashboard with trend analysis |
| **Continuously improve security posture against evolving threats** | Annual program audit; updated threat assessments |

## Benefits of Formal Governance

| Benefit | Without Governance (Current) | With Governance (Target) |
|---|---|---|
| **Risk Visibility** | Risks unknown and untracked | Formal risk register with ownership |
| **Accountability** | No one owns security decisions | CISO + risk owners with defined authority |
| **Investment Justification** | Security spending is ad hoc | Risk-based budget allocation with ROI tracking |
| **Regulatory Readiness** | No compliance validation | Audit-ready documentation and controls |
| **Incident Preparedness** | No response capability | Tested IR/DR/BC plans with defined escalation |
| **Stakeholder Trust** | Implicit trust in IT | Demonstrable security posture for partners and clients |

## Executive & Board Responsibilities: NACD Essential Practices

The **National Association of Corporate Directors (NACD)** defines essential practices for board-level cybersecurity oversight. Applied to CGT:

### Board-Level Responsibilities

| NACD Practice | CGT Implementation |
|---|---|
| **Cybersecurity as a regular board agenda item** | Quarterly cybersecurity briefing from CISO to CEO (and board if applicable) |
| **Appoint and support cybersecurity leadership** | CISO appointed, reporting to COO, with direct access to CEO for escalation |
| **Oversee policies and risk management** | Board/CEO reviews and approves EISP; CISO presents risk register quarterly |
| **Ensure appropriate resource allocation** | Security budget reviewed annually against risk assessment findings |
| **Understand the legal and regulatory landscape** | Regu-Nation software (existing) used to track compliance requirements; CISO provides regulatory updates |

### Executive Management Responsibilities

Executive management bridges strategic oversight and operational implementation:

| Responsibility | Owner |
|---|---|
| Implement cybersecurity strategy aligned with business goals and risk appetite | CEO + COO |
| Manage day-to-day cybersecurity operations including incident response | CISO |
| Allocate resources efficiently based on risk assessments | COO + CBO (budget) |
| Continuously monitor cybersecurity risks and performance metrics | CISO + Security Analyst |
| Maintain transparent communication with the board on cybersecurity status | CISO → COO → CEO |
| Promote a culture of security awareness across the organization | CISO + Awareness Coordinator + HR |
| Ensure compliance with regulations and internal policies | CISO + Policy & Risk Manager |

## Role of the CISO as Program Designer

The CISO is the **architect and operational leader** of CGT's cybersecurity program:

| CISO Responsibility | Deliverable |
|---|---|
| **Develop and align cybersecurity strategy** | Cybersecurity strategic plan aligned with CGT's product roadmap and business goals |
| **Build core programs** | Risk management program, incident response capability, compliance framework |
| **Communicate risk posture** | Quarterly executive dashboard; annual board presentation |
| **Lead training and awareness** | SETA program delivered through Awareness Coordinator |
| **Define and track metrics** | KPIs for risk reduction, incident response time, training completion, vulnerability remediation |
| **Drive continuous improvement** | Annual program audit; IDEAL model for maturity progression |

## Key Program Components & Frameworks

| Component | Framework | Purpose |
|---|---|---|
| **IT Governance** | COBIT / ITGI | Strategic alignment, resource management, performance measurement |
| **Risk Management** | NIST CSF | Identify → Protect → Detect → Respond → Recover |
| **Controls** | NIST SP 800-53 | Technical and administrative control selection |
| **Contingency Planning** | NIST SP 800-34 | IR, DR, BC planning and testing |
| **Continuous Improvement** | IDEAL Model | Initiating → Diagnosing → Establishing → Acting → Learning |

### IDEAL Model Application at CGT

| Phase | CGT Activity |
|---|---|
| **Initiating** | Establish CISO role, form cybersecurity unit, secure executive sponsorship |
| **Diagnosing** | Conduct risk assessment, BIA, policy gap analysis |
| **Establishing** | Develop policies, implement controls, build IR/DR/BC capability |
| **Acting** | Deploy SIEM, conduct training, execute remediation roadmap |
| **Learning** | Post-incident reviews, exercise AARs, annual program audit, feed lessons into next cycle |

## Governance Reporting Structure

```
Board / CEO
    ↑ Annual cybersecurity posture report
    ↑ Material incident notifications (immediate)
    │
COO (Executive Sponsor)
    ↑ Quarterly risk dashboard
    ↑ Budget and resource requests
    │
CISO (Program Owner)
    ↑ Monthly operational reports
    ↑ Incident reports (as needed)
    │
├── Policy & Risk Manager → Risk register, compliance status
├── Security Architect → Architecture decisions, control design
├── Security Engineer → Tool deployment, vulnerability status
├── Security Analyst → Monitoring alerts, incident triage
└── Awareness Coordinator → Training metrics, phishing simulation results
```
